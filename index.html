SQL + Python analysis

Brazilian E-Commerce Business Analysis

SQL SQLite Python Pandas Seaborn Jupyter
Dataset
Olist commerce
Lens
Delivery + reviews

A multi-table business analysis project using the Olist Brazilian E-Commerce dataset to study orders, delivery reliability, customer reviews, product categories, revenue, and regional performance.

Key result: Delivery delays were strongly associated with lower review scores, making logistics one of the clearest drivers of customer satisfaction.

  • Built a SQLite workflow across related CSV tables while checking table grain, keys, duplicates, and missing values.
  • Used SQL joins, grouped summaries, conditional logic, and business metrics to analyze delivery performance and customer satisfaction.
  • Created visual insights for monthly order trends, late delivery rates, review scores, state-level delivery issues, and category performance.
question.sql
SELECT review_score,
       COUNT(*) AS total_orders,
       ROUND(AVG(delivery_time_days), 2) AS avg_delivery_days,
       ROUND(AVG(is_late) * 100, 2) AS late_delivery_rate
FROM order_delivery_analysis
GROUP BY review_score
ORDER BY review_score;

Lichess Game Analysis

Dataset
200k games
Lens
Time pressure
♞

A 200,000-game Lichess analysis project studying how time controls, rating gaps, favorite reliability, and selected engine-evaluation features relate to online chess outcomes.

Key result: Bullet games were heavily shaped by time pressure, while rating gap was the clearest metadata signal for favorite reliability.

  • Used SQL and SQLite to analyze time-forfeit patterns, rating favorites, rating-gap buckets, and category-level differences.
  • Engineered evaluation features from messy wide columns while separating numeric engine evaluations from forced-mate notation.
  • Built focused visualizations showing that Bullet games are heavily shaped by time pressure, while rating gap is the clearest signal for favorite reliability.

Data analytics

Video Game Sales EDA

Python Pandas NumPy Matplotlib
Dataset
16.6k game records
Lens
Regional sales

My first full exploratory data analysis project, using a Kaggle dataset with around 16,600 game records to investigate sales patterns across platforms, genres, publishers, release years, and regions.

Key result: Platform, genre, publisher, and regional sales patterns varied strongly, but the dataset required careful limits because sales were aggregated by game rather than tracked over time.

  • Cleaned missing and inconsistent values while documenting dataset limitations.
  • Avoided misleading year imputation instead of forcing uncertain data into the analysis.
  • Built grouped summaries and visualizations for regional, genre, platform, and publisher patterns.
